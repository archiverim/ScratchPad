# 4 "L"s Retrospective
## Liked
- We were able to work under pressure and deliver a minimum viable product that had basic CRUD features and was somewhat aesthetically pleasing.
- We mostly followed our programming and communication standards to deliver consistent code styles throughout the codebase.
- We worked as a team to review merge requests and resolve any questions.

## Learned
- How to structure and start a JavaFX application UI.
- How to set realistic goals for a certain time frame, since we overestimated the amount of work we could complete.
- When to compromise code conciseness and minor styling differences in order to deliver a product by a deadline.

## Lacked
- A clear "branding" strategy. We should put more thought into logos, colour schemes and styling and define colours and other styling variables as constants in our CSS so they can be reused to enforce consistency with the design of the app.
- Sometimes our programming standards were not followed, regarding the naming standards of branches and commit messages as defined in our team working agreement.
- Due to scheduling conflicts, merge requests took a while to get in. We should specify a maximum time for which a merge request can sit pending before either merged or requested to be revised.

## Longed For
- Better and more frequent communication regarding being blocked by certain tasks, and check-ins stating what everyone is working on and when they are available.
- More clear assignment of tasks.  We should anaylze any tasks for a milestone and select them not only based on priority but also considering if they block any other tasks.
- Prioritizing completing those blocking tasks earlier in the sprint.
